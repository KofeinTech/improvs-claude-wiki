# Developer Rules

Non-negotiable rules for every developer at Improvs. Read all of these before your first commit.

## Rules

| Rule | One-liner |
|------|-----------|
| [Commit Only Yourself](rule-commit-only-yourself.md) | Never let AI auto-commit. Review every line. |
| [Small Tasks](rule-small-tasks.md) | Break work into 1-4 hour pieces. |
| [Plan Before Code](rule-plan-before-code.md) | Create a plan, review it, then implement. |
| [One Branch Per Task](rule-one-branch-per-task.md) | One Jira ticket = one branch = one PR. |
| [Separate Claude Contexts](rule-separate-claude-contexts.md) | Fresh session for each task. |
| [Review AI Critically](rule-review-ai-critically.md) | AI generates plausible code that may be wrong. |
| [Test Before PR](rule-test-before-pr.md) | Run tests and analyze before opening a PR. |
| [Use Shared Skills](rule-use-shared-skills.md) | Check for existing skills before writing custom prompts. |
| [Document Surprises](rule-document-surprises.md) | Write down non-obvious project knowledge. |
| [Report AI Gaps](rule-report-ai-gaps.md) | Report AI failures so they become shared skills. |
| [Communication](rule-communication.md) | Telegram for chat, Jira for decisions. |

## The golden rule

**AI drafts. You decide. You commit.**
