# Rule: Communication Channels

Use the right channel for the right information. Nothing actionable should live only in Telegram.

## Channel usage

| Information type | Where it goes |
|---|---|
| Technical decisions | Jira ticket comment |
| Architecture or process choices | Wiki page |
| Bug reports | Jira ticket |
| Task assignments | Jira ticket |
| Quick questions, informal chat | Telegram |
| Meeting notes with action items | Jira tickets (one per action item) |

## Why

Telegram messages get buried in hours. Jira tickets are searchable, assignable, and tracked. Wiki pages are the single source of truth for how things work. If a decision isn't in Jira or the wiki, it effectively doesn't exist.

## Golden rule

If someone asks "where was that decision made?" the answer should always be a Jira ticket or wiki link, never "scroll up in Telegram."
