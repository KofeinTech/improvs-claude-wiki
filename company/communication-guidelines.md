# Communication Guidelines

## Where to communicate what

| Type of message | Where | Why |
|----------------|-------|-----|
| Quick question, casual chat | Telegram | Fast, informal |
| Technical decision | Jira ticket comment | Searchable, linked to task, permanent |
| Architecture or process choice | Wiki page | Accessible to everyone, survives time |
| Bug report | Jira ticket | Tracked, assigned, prioritized |
| Task assignment or request | Jira ticket | Tracked, estimated, scheduled |
| Code discussion | GitHub PR comment | Linked to the code, reviewable |
| Client status update | Jira + email | Formal, documented |

## The rule

**If it's not in Jira or wiki, it doesn't exist.**

Telegram is for quick chat. Anything actionable, any decision, any assignment -- must be in Jira or wiki. When someone new joins the team or picks up a project, they should be able to understand everything from Jira and wiki without reading Telegram history.

## Response expectations

- Telegram: respond within a few hours during work time
- Jira comments: respond within 1 business day
- PR reviews: complete within 1 business day
- Urgent production issues: Telegram + Jira ticket immediately
